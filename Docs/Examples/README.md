# How to write an example

An example file (ie. print_hello_world.prn) should be named exactly
based on what it does, with clarity.

- Documentation is permitted as long as its using proper commenting
semantics.
- Each example file should be allowed to be directly ran by
the interpreter.
- No extra irrelevant logic should occur during the example file.
- The result should be immediately clear to the user who runs the
example.

# Naming semantics

The first part of the filename should explain the result:

- error_*
- print_*
- valid_*
- invalid_*

etc.

The remainder of the filename should be dedicated to explaining
exactly what the file will do.

- print_through_alphabet.prn
- print_hello_world.prn
- error_invalid_char.prn

etc.
